# ESP32S3-OV5640 WebSocket 广播对接说明

本文档面向接收端开发者，说明如何通过 WebSocket 接收设备主动推送的 JPEG 照片。

## 1. 总览

设备行为（当前版本）:

- ESP32 每隔 3 秒拍 1 张 JPEG。
- 新照片写入内存环形队列（容量 6 张）。
- 每次拍照成功后，服务端会向所有已连接的 WebSocket 客户端广播该最新照片（二进制帧）。

结论:

- 这是“推模式”对接，客户端无需轮询拉图。
- 客户端只需保持 WebSocket 长连接并处理二进制 JPEG 数据。

## 2. 连接前准备

1. 确保 ESP32 与接收端在同一网络且可互通。
2. 通过串口日志确认设备 IP。
3. 浏览器打开 `http://设备IP/`，确认服务在线。

## 3. 接口定义

### 3.1 WebSocket 照片广播（主接口）

- 协议: WS
- 地址: `ws://设备IP/ws`
- 数据方向:
  - 客户端 -> 设备: 文本命令（可选）
  - 设备 -> 客户端: JPEG 二进制

客户端命令:

- `latest`: 立即请求返回一张当前最新照片（用于刚连接时先拿首帧）

服务端推送:

- 每当有新照片写入，服务端广播一个 `HTTPD_WS_TYPE_BINARY` 帧。
- 帧载荷为完整 JPEG 字节流，可直接解码显示或落盘。

### 3.2 HTTP 页面（调试/演示）

- 方法: GET
- 路径: `/`
- 返回: HTML 页面
- 页面行为:
  - 自动连接 `/ws`
  - 默认发送一次 `latest`
  - 后续等待服务端广播并实时更新图片

### 3.3 兼容接口（可选）

- `GET /photo?id={id}`: 按 ID 下载单张 JPEG
- `GET /delete?id={id}`: 删除指定照片
- `GET /delete_all`: 删除全部照片

## 4. 客户端接入示例（浏览器 JS）

```javascript
const ws = new WebSocket("ws://192.168.1.10/ws");
ws.binaryType = "arraybuffer";

ws.onopen = () => {
  ws.send("latest");
};

ws.onmessage = (ev) => {
  if (typeof ev.data === "string") {
    console.log("text:", ev.data);
    return;
  }
  const blob = new Blob([ev.data], { type: "image/jpeg" });
  const url = URL.createObjectURL(blob);
  document.getElementById("img").src = url;
};
```

## 5. 时效与缓冲说明

当前参数:

- 拍照周期: 3 秒/张
- 队列容量: 6 张

理论缓存窗口约 18 秒。WebSocket 推送模式下，正常在线客户端通常不会错过新帧；若客户端断线重连，建议连接成功后发送 `latest` 立即补一帧。
