# ESP32S3-OV5640 WebSocket 推流对接说明

本文档说明当前固件的唯一对接方式: WebSocket 推流。

## 1. 当前能力

- 设备每次拍照成功后，会向所有 WebSocket 客户端广播一帧 JPEG 二进制。
- 浏览器演示页保留在 `/`，仅用于显示 `/ws` 推流画面。
- 不再提供历史图片下载、删除等兼容 HTTP 接口。
- 当前固件已配置固定地址: `192.168.1.188:80`。

## 2. 接口定义

### 2.1 WebSocket 主接口

- 协议: WS
- 固定地址: `ws://192.168.1.188/ws`
- 服务端推送: JPEG 二进制帧

可选客户端命令:

- `latest`: 客户端连接成功后可发送一次，立即请求当前最新帧。

### 2.2 演示页面

- 方法: GET
- 路径: `/`
- 作用: 显示 WebSocket 实时画面
- 固定访问地址: `http://192.168.1.188/`

## 3. 已删除接口

以下接口已移除，请勿再调用:

- `GET /photo?id={id}`
- `GET /delete?id={id}`
- `GET /delete_all`

## 4. Python 接收端脚本

仓库内已提供脚本: `tools/ws_receiver.py`

### 4.1 安装依赖

```bash
pip install websockets
```

### 4.2 启动接收

```bash
python tools/ws_receiver.py --out received
```

说明:

- 默认固定连接 `ws://192.168.1.188:80/ws`
- 收到的 JPEG 将保存到 `received/` 目录
- 断线会自动重连

仅看接收速率不落盘:

```bash
python tools/ws_receiver.py --no-save
```

## 5. 对接建议

- 接收端应长期保持 WebSocket 连接。
- 连接建立后发送一次 `latest`，避免等待下一次拍照周期。
- 若你要做业务处理，建议在客户端按时间戳或序号自行落盘与索引。
- 如果你的路由网段不是 `192.168.1.x`，请同步修改 `src/main.cpp` 的静态 IP 参数与 `tools/ws_receiver.py` 的固定地址常量。
