# ESP32S3-OV5640 照片接收端对接说明

本文档面向“接收方系统”开发者，目标是让你可以稳定接收设备拍摄的 JPEG 照片。

## 1. 总览

设备行为（当前版本）:

- ESP32 每隔 3 秒拍 1 张 JPEG。
- 照片存放在设备内存环形队列，容量为 6 张。
- 提供 HTTP 接口供接收端拉取图片。

结论:

- 这是一种“拉模式”对接，不是设备主动推送。
- 接收端需要轮询设备并按照片 ID 拉取新图。
- 由于环形覆盖，旧图会被新图覆盖，接收端必须及时拉取。

## 2. 连接前准备

1. 确保 ESP32 和接收端在同一网络可互通。
2. 通过串口日志获取设备 IP。
3. 访问以下地址验证连通性:
   - http://设备IP/

串口中会打印类似信息:

- WiFi 连接成功
- ESP32 IP 访问地址: 192.168.x.x

## 3. 接口定义（当前固件）

### 3.1 获取照片列表页

- 方法: GET
- 路径: /
- 返回: text/html
- 用途: 页面中包含所有可下载照片链接

示例片段（结构说明）:

- /photo?id=123
- frame_123.jpg
- ts=时间戳毫秒
- /delete?id=123

说明:

- 列表按时间倒序展示（新图在前）。
- 这是 HTML 页面，不是 JSON。

### 3.2 下载单张照片

- 方法: GET
- 路径: /photo?id={id}
- 入参:
  - id: 正整数，必填
- 成功返回:
  - HTTP 200
  - Content-Type: image/jpeg
  - Body: JPEG 二进制
- 失败返回:
  - HTTP 400: 缺少 id 或 id 非法
  - HTTP 404: 对应 id 已不存在（被覆盖或删除）




## 4. 环形覆盖与时效要求

当前参数:

- 拍照周期: 3 秒/张
- 内存容量: 6 张

理论可回溯窗口约 18 秒（6 x 3 秒）。

这意味着:

- 如果接收端中断超过约 18 秒，最早的未拉取图片可能已被覆盖。
- 覆盖后再请求旧 ID，会得到 404。
